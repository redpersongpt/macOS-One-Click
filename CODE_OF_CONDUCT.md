# Code of Conduct

## Our Standard

This project is for people who want to build, debug, and improve OpCore-OneClick without turning the repo into a hostile place to work.

Be direct. Be technical. Be respectful.

We expect contributors to:

- Assume good faith unless there is clear evidence otherwise
- Keep criticism focused on code, behavior, decisions, and evidence
- Share reproduction steps, logs, and context when reporting problems
- Respect maintainers' time by reading existing docs and issue history first
- Avoid harassment, personal attacks, discrimination, or intimidation

## Unacceptable Behavior

The following is not acceptable in project spaces, issues, pull requests, discussions, or direct contributor interactions:

- Harassment, threats, or abusive language
- Personal insults or repeated hostile behavior
- Discriminatory remarks or hate speech
- Publishing private information without consent
- Spam, trolling, or deliberately derailing technical threads
- Repeated bad-faith issue filing after maintainers have closed the matter

## Enforcement

Maintainers may remove comments, close threads, reject contributions, or block participants when behavior crosses the line.

Responses depend on severity and pattern. Not every case gets a warning first.

## Reporting

If you need to report conduct problems, open a private GitHub report to the maintainer or use the contact route listed in [SECURITY.md](SECURITY.md) if that is the safest option available.

## Scope

This code of conduct applies to:

- GitHub issues
- Pull requests
- Discussions
- Project documentation
- Any other official project communication channels
